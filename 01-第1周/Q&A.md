## Q1：Python中 is 和 == 的区别
- **`is` 比较的是“内存地址”**
- **` == ` 比较的是“值（内容）”**：判断两个变量里的数据相不相等。
在实际开发中，`is` 最标准、最常用的场景是**与单例对象（Singleton）进行比较**，尤其是 `None`。
	比较**是不是同一个对象实例**，或者判断**是否为 `None`、`True`、`False`** 用 `is`。
## Q2: del的操作
**`del` 删除的是变量的“标签（引用）”，而不是直接销毁内存中的数据本身。**（内存的清理是由 Python 的垃圾回收机制自动完成的）
## Q3: types类型的编码和解码
**`str` 变成 `bytes` 叫编码 (`encode`)**
**`bytes` 变成 `str` 叫解码 (`decode`)**
用什么格式（如 utf-8）编码，就必须用什么格式解码
**对 `bytes` 进行索引取值时，得到的是一个整数 (0-255)**。
## Q4: repr（）函数
- **`str()` 的目标是“可读性”（给用户看）**：它尽可能把数据转成人类最容易看懂的排版格式。
- **`repr()` 的目标是“准确无歧义”（给程序员/机器看）**：它会显示这个数据在 Python 底层到底长什么样，带有数据类型的特征。
**`repr()` 返回的字符串，最好能够是一段合法的 Python 代码。**
## Q5: 字典的深拷贝和浅拷贝
在 Python 中，如果直接用 ` = ` 把一个字典赋给另一个变量，它们实际上**指向同一个内存地址**（相当于起了个别名）。修改其中一个，另一个也会跟着变。
使用 `dict.copy()` 可以解决这个问题，但是是浅拷贝：**如果字典里面的值（value）是列表或嵌套字典等“可变对象”**，那么新字典和老字典里的这个列表依然指向同一个内存地址。
深拷贝：`deepcopy()`
