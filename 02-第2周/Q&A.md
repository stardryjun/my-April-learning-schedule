## Q1：已经有实例方法，类方法的作用是什么
1. 使用类方法，操作类属性更加语义化，显示我们的意图是操作类级别的状态。
2. 类方法经常用来创建类的**辅助构造方法**，提供一些额外方式（**工厂方法**）来实例化类，而不是直接使用 `__init__`。类方法可通过 `cls` 调用类的构造方法 `__init__`，使函数通用性更强。如果类被子类化，类方法中的 `cls` 确保创建的对象是子类实例，而不是强制返回父类实例。
3. 在继承结构中，类方法的出现确保了方法调用是**绑定到调用类本身**的，而不是硬绑定到父类。
## Q2: 什么是闭包
闭包是 **一个嵌套函数**，并且这个嵌套函数**可以使用它外部函数的变量**，即使外部函数已经执行结束，那些变量也依然存在，不会被销毁
如果想在内部函数中**修改**外部变量，需要用 `nonlocal` 声明
装饰器就是闭包！
闭包的优势：
- 变量私有化：
- 闭包将数据封装在一个函数内部，数据只对这个函数可见。
- 不需要依赖全局变量，从而减少了潜在的名字冲突，数据更加安全。
## Q3: 类装饰器与普通装饰器的区别
类装饰器可以是函数，也可以是类。
当函数作为类装饰器，函数接受一个类作为参数(cls)，并返回类；
当类作为类装饰器，类装饰器定义`__init__`和`__call__`
```
class ClassDecorator:
    def __init__(self, cls):  # **1. 这是装饰器类的初始化**
        self.cls = cls  # 保存被装饰的类
    def __call__(self, *args, **kwargs):  # **2. 这是装饰器类的核心，负责控制被装饰类的实例化**
        instance = self.cls(*args, **kwargs)  # 创建原来的类的实例
        instance.new_method = lambda: print("This is a new method")  # 动态添加新方法
        return instance  # 返回被增强的实例

@ClassDecorator
def SomeClass:
# 即 SomeClass = ClassDecorator(SomeClass)
# 变量名 `SomeClass` 不再指向原始类，而是指向装饰器实例。装饰器实例中保存着原始类 (`self.cls`)。
```
`__init__(self,cls):`当创建 `ClassDecorator` 的实例时，`__init__` 方法会自动执行，用来“初始化”这个装饰器。
- **`cls` 其实就是我们想要增强的类**，比如 `SomeClass`。
- `self` 代表当前的装饰器实例。
- `self.cls = cls` 的作用？**把目标类保存下来**
`__call__(self,*args,**kargs):`调用装饰器时，**决定被装饰的类应该怎么被增强**。
`instance = self.cls(*args, **kwargs)`
- `self.cls` 是我们之前保存在 `__init__` 中的目标类。
- `self.cls(*args, **kwargs)` 就等效于调用目标类的构造方法，生成一个它的实例。即`SomeClass(*args,**kwargs)`
## Q4: class A(B):
`class A(B):`代表继承关系，而不是函数调用关系
没有class时，A(B)代表调用`__call__`
- 如果 `A` 没有定义 `__call__`：直接调用 `A(arg)` **会报错**，因为这个实例不支持调用语法。
## Q5: 为什么迭代器的__iter__返回自身？
- 当我们调用 `iter(obj)` 方法时，它会调用对象的 `__iter__` 方法，期望获得一个迭代器。
- 如果对象本身已经是迭代器（即实现了 `__iter__` 和 `__next__`），那么 `__iter__()` 就直接返回自己是最自然且高效的设计。
## Q6:自己设计的迭代器如果需要重复利用
标准的迭代器协议设计中，迭代器是"用完即丢"的，无法重置。
可以在next使用完后要恢复为初始值或在iter中设置每次调用时将迭代器的状态重置；
可以每次创建新的迭代器实例；
可以使用生成器，每次调用函数都会创建**新的生成器**；
