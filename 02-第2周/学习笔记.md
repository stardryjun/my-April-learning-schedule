### 类
类是模板，对象是具体的实现。
#### 属性
类属性：属于类本身，所有对象共享。
- 所有实例共享类属性，当类属性发生变化时，这种更改会影响所有实例（除非实例覆盖它）。
实例属性：属于对象，由每个对象单独拥有。
- 在对象的 `__init__` 方法中或通过 `self` 定义
- 如果实例属性和类属性同名，**实例属性会覆盖类属性**
#### 方法
##### 实例方法
- 第一个参数总是 `self`，表示该方法属于当前对象。
- 可以通过 `self` 访问实例的属性和其他方法。
- `self` 不是 python 关键字
##### 类方法
- 类方法的第一个参数是 `cls`  ，表示当前的类本身（而不是具体的实例）。
- 通过 `@classmethod` 装饰器定义。
- 类方法通常用于操作和修改类属性，而不依赖于具体的实例。
##### 静态方法
- 静态方法是与类或实例**都无关的方法**，它不需要访问类（`cls`）或实例（`self`）。
- 通过 `@staticmethod` 装饰器定义。
- 它通常用于实现工具逻辑，逻辑往往与类的外部上下文有关，但将逻辑置于类的作用范围内有助于组织代码结构。
#### 特殊方法（魔术方法）
- **`__init__`**：初始化对象时自动调用。
- **`__str__`**：用于打印对象时自动调用。
- **`__repr__`**：对象字符串表示（调试用）。
- **`__del__`**：对象销毁时自动调用。
- **`__call__`**：对象可被当作函数调用

- 大多被 Python 内部调用，**不需要直接调用**。

###### `__call__`
- 定义了 `__call__` 方法后，可以将对象 `obj` 当作函数直接调用。
- 调用时传入的参数会被传递给 `__call__`。
- 装饰器本质上是一个函数增强工具，`__call__` 可以简化装饰器的实现。
- `__call__` 适合用于实现缓存机制或惰性计算。
###### **`__enter__` / `__exit__`**
与 `with` 语句配合使用
**`__enter__(self)`**
- 它的返回值赋值给 `as` 后的变量（如果存在）。
**`__exit__(self, exc_type, exc_value, traceback)`**
- 接收异常信息（如果 `with` 块引发了异常），并可以选择处理这些异常。
- `exc_type`: 异常类型 (e.g., ValueError, TypeError)
- `exc_value`: 异常实例 (e.g., ValueError("An error occurred"))
- `traceback`: 异常的追溯信息
- 如果 `__exit__` 返回 `True`，异常将被抑制；否则，异常会在 `with` 块外再次引发。
###### `__name__`
用于表示当前模块的名称
if __name__ == "__main__":
- 当模块是被直接运行时，`__name__` == `"__main__"`，它会执行代码块中的逻辑。
- 当模块是被导入时，`__name__` 是模块名，代码块不会被运行。
###### 上下文管理器
上下文管理协议指的是实现 `__enter__` 和 `__exit__` 两个方法的对象。这些对象称为**上下文管理器（Context Manager）**。
内置 `open` 函数的上下文管理支持 `__enter__` 和 `__exit__`，可以自动关闭文件。

上下文管理器非常适合管理需要显式清理的资源，如：
- 文件操作 (`open`)
- 数据库连接
- 网络套接字
- 多线程/多进程的锁
Python 标准库提供了 `contextlib` 模块，允许快速实现上下文管理器。
利用 `@contextmanager` 装饰器，你可以用函数快速创建上下文管理器。
###### `__hash__`
定义一个对象是否可以作为**哈希表数据结构**的键（例如 `set` 和 `dict` 的键）或是否能够参与哈希值计算（`hash()` 函数）。
- **可哈希对象：** 要求实现 `__hash__` 方法，并且是不可变的（不可变类型的实例，如元组和数值类型，为天然可哈希类型）。
- **不可哈希对象：** 默认情况下，可变对象（如列表、字典）是不可哈希的，因为它们的值可能发生变化，无法保证哈希表的一致性。
`__hash__` 方法允许自定义对象的哈希行为。
- **返回值：** 必须返回一个整数，作为对象的哈希值。
1. **两个相等（`__eq__` 完全相等）的对象，其哈希值必须相同。**
2. **不同对象的哈希值尽可能不同，避免冲突**。
- **对于用户自定义类**： 默认情况下，`__hash__` 的行为是基于对象的标识（`id()` 函数的返回值），两个不同对象即使内容一样，其哈希值也不同。
- 如果定义了 `__eq__`，但没有定义 `__hash__`，Python 会禁止该对象作为哈希值的键，原因是哈希值和相等性没有保持一致性。
**最佳实践**要求，哈希值依赖的属性**不应改变**（即使是设计上的约束）。否则，哈希表（例如字典或集合内部）会因哈希值变化而导致数据问题。
哈希冲突不会导致 `dict` 或 `set` 错误，只是性能会有所降低。

如果你只定义了 `__eq__`，没有定义`__hash__`Python 会抑制（禁用）对象的 `__hash__` 方法。
如果你只定义了 `__eq__` 且不关心对象是否可哈希，你可以明确声明 `__hash__` = `None`，告知 Python 禁用哈希功能。
有时，你定义了 `__eq__`，但仍需要使用默认的哈希行为。可以通过将 `__hash__` 指向 `object.__hash__` 来启用哈希。

- 使用 `@property` 将类的方法“伪装”成只读**属性**。

## 装饰器、生成器、迭代器、上下文管理器（with）
### 装饰器
`@decorator_name`
内置的装饰器
	`@staticmethod`、`@classmethod`定义静态方法和类方法。
**日志记录**、**性能分析**、**权限控制**、**缓存**
```
@decorator_function
def original_function():
    pass
```
等价于
```
def original_function():
    pass
original_function = decorator_function(original_function)  # 装饰器的效果
```
如果原函数需要参数，可以在装饰器的 wrapper 函数中传递参数：
```
def simple_decorator(func):
	def wrapper(*args, **kwargs):
		print("装饰器功能: 函数执行前输出这句话")
		func(*args, **kwargs) # 执行原始函数
		print("装饰器功能: 函数执行后输出这句话")
	return wrapper
```
如果需要给装饰器传递参数，则需要再嵌套一层函数。
一个函数可以叠加多个装饰器，它们按照**就近原则**从上到下依次执行，而被装饰的函数从内向外调用。
装饰器会改变原函数的一些属性，例如名称 `__name__` 和文档字符串 `__doc__`，导致函数元信息被替换成装饰器中返回的 `wrapper` 函数的信息。
	解决方法：`functools.wraps` 装饰器
#### *类装饰器（未懂）*
一个类装饰器通常至少包含两个部分：
1. `__init__` 方法：初始化并保存要装饰的类。
2. `__call__` 方法：对目标类进行修改或增强，返回一个新对象或调用目标类实例化。
```
class DecoratorClass:
    def __init__(self, cls):
        self.cls = cls
    def __call__(self, *args, **kwargs):
        # 修改原始类的行为，或者添加功能
        instance = self.original_class(*args, **kwargs)
        return instance
```
#### 内置装饰器
1. **`@staticmethod`**: 将方法定义为静态方法，不需要实例化类即可调用。
2. **`@classmethod`**: 将方法定义为类方法，第一个参数是类本身（通常命名为 `cls`）。
3. **`@property`**: 将方法转换为属性，使其可以像属性一样访问。
### 迭代器
迭代器是一个可以记住遍历的位置的对象。
迭代器有两个基本的方法：**iter()** 和 **next()**。
把一个类作为一个迭代器使用需要在类中实现两个方法 __iter__() 与 __next__() 。
StopIteration 异常用于标识迭代的完成，防止出现无限循环的情况
- **惰性计算**：每次仅在需要时生成下一个值，不会一次性占用大量内存。
- **一次性的**：迭代器只能遍历一次数据，不能重复遍历。

可迭代对象：实现了`__iter__()`方法的对象
迭代器对象：实现了`__iter__()`和`__next__()`方法的对象称为迭代器对象

`__iter__()`方法要返回一个迭代器对象，你可以使用iter() 函数来触发它。每次调用可迭代对象的`__iter__()`，都返回一个**新的迭代器对象**。
`__next__()`方法用于从迭代器中返回下一个值，如果没有可返回值了，抛出 `StopIteration`异常。你可以使用next() 函数来触发它。

迭代器的 __iter__ 返回的是自身
### 生成器
使用了 **yield** 的函数被称为生成器
当在生成器函数中使用 **yield** 语句时，函数的执行将会暂停，并将 **yield** 后面的表达式作为当前迭代的值返回。
然后，每次调用生成器的 **next()** 方法或使用 **for** 循环进行迭代时，函数会从上次暂停的地方继续执行，直到再次遇到 **yield** 语句。
#### 生成器表达式
`(expression for item in iterable if condition)`

生成器可以产生无限序列，这在普通列表或元组中无法实现。

`send()`
- 使用 `send(value)` 可以将值发送到生成器的 `yield` 表达式位置。
- 一般在需要双向通信的时候用到。
`close()`
- 调用 `close()` 方法可以结束一个生成器，后续调用会抛出 `StopIteration`。
### 上下文管理器
上下文管理器允许用户**定义并控制代码块的进入与退出操作**，在代码块执行时保证资源正确分配和释放。
实现：
- `__enter__()`：进入上下文时的初始化逻辑，返回上下文的资源对象。
- `__exit__(exc_type, exc_value, traceback)`：退出上下文时的清理逻辑。

Python 的 `contextlib` 模块提供了装饰器 `@contextmanager`，将一个普通的**生成器函数**转换为上下文管理器。
`yield` 之前的代码对应于 `__enter__`
`yield` 之后的代码对应于 `__exit__`

`contextlib.closing` 将实现了 `close()` 方法的对象包装成一个上下文管理器。

### os
提供与操作系统交互的功能。
`os` 模块是跨平台的，这意味着你可以在不同的操作系统（如 Windows、Linux、macOS）上使用相同的代码。

3. 列出目录内容`os.listdir(path)`
4. 创建目录`os.mkdir(path)` `os.makedirs("nested/dir/structure")`
5. 删除目录`os.rmdir(path)` `os.removedirs("nested/dir/structure")`
