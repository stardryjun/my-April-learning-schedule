### 类
类是模板，对象是具体的实现。
#### 属性
类属性：属于类本身，所有对象共享。
- 所有实例共享类属性，当类属性发生变化时，这种更改会影响所有实例（除非实例覆盖它）。
实例属性：属于对象，由每个对象单独拥有。
- 在对象的 `__init__` 方法中或通过 `self` 定义
- 如果实例属性和类属性同名，**实例属性会覆盖类属性**
#### 方法
##### 实例方法
- 第一个参数总是 `self`，表示该方法属于当前对象。
- 可以通过 `self` 访问实例的属性和其他方法。
- `self` 不是 python 关键字
##### 类方法
- 类方法的第一个参数是 `cls`  ，表示当前的类本身（而不是具体的实例）。
- 通过 `@classmethod` 装饰器定义。
- 类方法通常用于操作和修改类属性，而不依赖于具体的实例。
##### 静态方法
- 静态方法是与类或实例**都无关的方法**，它不需要访问类（`cls`）或实例（`self`）。
- 通过 `@staticmethod` 装饰器定义。
- 它通常用于实现工具逻辑，逻辑往往与类的外部上下文有关，但将逻辑置于类的作用范围内有助于组织代码结构。
#### 特殊方法（魔术方法）
- **`__init__`**：初始化对象时自动调用。
- **`__str__`**：用于打印对象时自动调用。
- **`__repr__`**：对象字符串表示（调试用）。
- **`__del__`**：对象销毁时自动调用。
- **`__call__`**：对象可被当作函数调用

- 大多被 Python 内部调用，**不需要直接调用**。

###### `__hash__`
定义一个对象是否可以作为**哈希表数据结构**的键（例如 `set` 和 `dict` 的键）或是否能够参与哈希值计算（`hash()` 函数）。
- **可哈希对象：** 要求实现 `__hash__` 方法，并且是不可变的（不可变类型的实例，如元组和数值类型，为天然可哈希类型）。
- **不可哈希对象：** 默认情况下，可变对象（如列表、字典）是不可哈希的，因为它们的值可能发生变化，无法保证哈希表的一致性。
`__hash__` 方法允许自定义对象的哈希行为。
- **返回值：** 必须返回一个整数，作为对象的哈希值。
1. **两个相等（`__eq__` 完全相等）的对象，其哈希值必须相同。**
2. **不同对象的哈希值尽可能不同，避免冲突**。
- **对于用户自定义类**： 默认情况下，`__hash__` 的行为是基于对象的标识（`id()` 函数的返回值），两个不同对象即使内容一样，其哈希值也不同。
- 如果定义了 `__eq__`，但没有定义 `__hash__`，Python 会禁止该对象作为哈希值的键，原因是哈希值和相等性没有保持一致性。
**最佳实践**要求，哈希值依赖的属性**不应改变**（即使是设计上的约束）。否则，哈希表（例如字典或集合内部）会因哈希值变化而导致数据问题。
哈希冲突不会导致 `dict` 或 `set` 错误，只是性能会有所降低。

如果你只定义了 `__eq__`，没有定义`__hash__`Python 会抑制（禁用）对象的 `__hash__` 方法。
如果你只定义了 `__eq__` 且不关心对象是否可哈希，你可以明确声明 `__hash__` = `None`，告知 Python 禁用哈希功能。
有时，你定义了 `__eq__`，但仍需要使用默认的哈希行为。可以通过将 `__hash__` 指向 `object.__hash__` 来启用哈希。

#### 类装饰器
一个类装饰器通常至少包含两个部分：
1. `__init__` 方法：初始化并保存要装饰的类。
2. `__call__` 方法：对目标类进行修改或增强，返回一个新对象或调用目标类实例化。
```
class DecoratorClass:
    def __init__(self, cls):
        self.cls = cls
    def __call__(self, *args, **kwargs):
        # 修改原始类的行为，或者添加功能
        instance = self.original_class(*args, **kwargs)
        return instance
```

if `decorator` is a class, it needs to take `func` as an argument in its [`.__init__()` initializer](https://realpython.com/python-class-constructor/). Furthermore, the class instance needs to be [callable](https://realpython.com/python-callable-instances/) so that it can stand in for the decorated function.
For a class instance to be callable, you implement the special [`.__call__()`](https://docs.python.org/3/reference/datamodel.html#emulating-callable-objects) method. Therefore, a typical implementation of a decorator class should implement `.__init__()` and `.__call__()`
Note that you need to use the `functools.update_wrapper()` function instead of `@functools.wraps`.
##### `__new__`
A subtle and important detail to note about `.__new__()` is that it can also return an instance of a class different from the class that implements the method itself. When that happens, Python doesn’t call `.__init__()` in the current class, because there’s no way to unambiguously know how to initialize an object of a different class.
Now, if you need a custom implementation of this method, then you should follow a few steps:
1. **Create a new instance** by calling `super().__new__()` with appropriate arguments.
2. **Customize the new instance** according to your specific needs.
3. **Return the new instance** to continue the instantiation process.
```
class SomeClass:
    def __new__(cls, *args, **kwargs):
        instance = super().__new__(cls)
        # Customize your instance here...
        return instance #return to __init__
```
 You should **always** define `.__new__()` with `*args` and `**kwargs`, unless you have a good reason to follow a different pattern.
`object.__new__()` is the base implementation of `.__new__()` for all Python classes.
 Finally, in the third step, you need to **return** the new instance to continue the instantiation process with the initialization step.
 It’s important to note that `object.__new__()` itself only accepts a _single_ argument, the class to instantiate.
 you can use `.__new__()` to create subclasses of [immutable](https://realpython.com/python-mutable-vs-immutable-types/) types, such as [`int`](https://realpython.com/python-numbers/#integers), [`float`](https://realpython.com/python-numbers/#floating-point-numbers), [`tuple`](https://realpython.com/python-lists-tuples/), and [`str`](https://realpython.com/python-strings/).

`import sys`
`sys.argv` 是一个包含命令行参数的列表。`sys.argv[0]` 是脚本的名称，后续元素是传递给脚本的参数。
`sys.exit()` 用于退出程序。你可以传递一个整数作为退出状态码，通常 `0` 表示成功，非零值表示错误。
`sys.stdin`、`sys.stdout` 和 `sys.stderr` 分别代表标准输入、标准输出和标准错误流。你可以重定向这些流以实现自定义的输入输出行为。
`sys.version` 和 `sys.version_info` 提供了当前 Python 解释器的版本信息。
`sys.path` 是一个列表，包含了 Python 解释器在导入模块时搜索的路径。你可以修改这个列表来添加自定义的模块搜索路径。
`sys.modules` 是一个字典，记录了所有当前已加载的模块。当你导入模块时，Python 会先检查该模块是否已存在于 `sys.modules` 中，若已存在，则直接返回模块引用，而不会重复加载。
`sys.platform` 表示当前操作系统的标识符。
`sys.getsizeof()`返回对象在内存中的字节大小。
`sys.maxsize`返回 Python 整数（`int` 类型）支持的最大值。
`sys.exc_info()`返回当前线程的异常信息。
### json
JSON (JavaScript Object Notation) 是一种轻量级的数据格式，广泛用于网络数据传输和存储。
`import json`
`json.dumps(obj, *, skipkeys=False, ensure_ascii=True, indent=None)`
- `obj`：要转换的 Python 对象。
- `ensure_ascii`：默认是 `True`，将非 ASCII 字符转义为 `\u` 的 Unicode 格式；如果为 `False`，直接输出非 ASCII 字符。
- `indent`：格式化美化 JSON 字符串输出（指定缩进级别）。
- `skipkeys`：是否跳过非字符串键（如 `int` 类型键）
`json.dump(obj, fp, *, ensure_ascii=True, indent=None)`
- `fp`：输出目标文件对象。
`json.loads(s, *, parse_float=None, parse_int=None)`
`json.load(fp)`

|**Python Type**|**JSON Type**|
|---|---|
|`dict`|`object`|
|`list`, `tuple`|`array`|
|`str`|`string`|
|`int`, `float`|`number`|
|`True` / `False`|`true` / `false`|
|`None`|`null`|

对于无法直接 JSON 序列化的对象（如类实例），需要自定义逻辑。
`object_hook`用于将 JSON 字典解码为自定义类实例
### datetime
`import datetime`
当前日期和时间
- `datetime.now()`：返回当前系统的日期和时间。
- `datetime.today()`：返回当前的日期和时间（几乎等同于 `now()`）。
- `datetime.utcnow()`：返回当前 UTC 时间（格林威治标准时间）。
指定日期和时间
- 使用 `datetime(year, month, day, hour, minute, second, microsecond)` 初始化对象
使用 `strftime` 方法将 `datetime` 对象格式化为特定格式的字符串。
- 常用格式符号：
    - `%Y`：四位年份。
    - `%m`：两位月份。
    - `%d`：两位日期。
    - `%H`：两位小时（24 小时制）。
    - `%M`：两位分钟。
    - `%S`：两位秒。
    - `%A`：星期的全名。
    - `%a`：星期的简称。
使用 `strptime` 方法将符合格式的时间字符串解析为 `datetime` 对象。

- `date.today()` 返回当前日期（不包含时间）。
- 使用 `date(year, month, day)` 创建一个日期对象。
- `time(hour, minute, second, microsecond)` 用于表示时间
- `timedelta` 是用来表示**两个日期时间之间的差值（时间间隔）**的类。

Python 的 `datetime` 支持时区处理，但操作起来较复杂，通常依赖外部包 `pytz` 或者新版的 `timezone`。
### re
`re` 是 Python 中用于处理正则表达式的模块。
`import re`

`re.match()`从**字符串开始**的位置匹配正则表达式。`re.match(pattern, string, flags=0)`
- `pattern`：正则表达式模式。
- `string`：待匹配的字符串。
- `flags`：可选，用于指定额外的匹配规则（如忽略大小写）。
`re.search()`搜索整个字符串，返回第一个匹配的结果。
- 区别：
    - `re.match()` 只在字符串的开头尝试匹配。
    - `re.search()` 会扫描整个字符串直到找到匹配。
`re.findall()`返回字符串中所有非重叠匹配项的列表。`re.findall(pattern, string, flags=0)`
`re.split()`使用匹配结果分割字符串，返回分割后的列表。`re.split(pattern, string, maxsplit=0, flags=0)`
`re.sub()`替换字符串中匹配项的内容。`re.sub(pattern, repl, string, count=0, flags=0)`
`re.compile()`预编译正则表达式以提升匹配效率。`re.compile(pattern, flags=0)`

**编译标志（`flags` 参数）**

|**标志**|**描述**|
|---|---|
|`re.I` / `re.IGNORECASE`|忽略大小写。|
|`re.M` / `re.MULTILINE`|多行模式，`^` 和 `$` 匹配每行的起始和结束位置。|
|`re.S` / `re.DOTALL`|使 `.` 匹配所有字符，包括换行符。|
|`re.X` / `re.VERBOSE`|允许正则表达式以更易读的格式书写，可以包含注释和空格。|
### collections
`collections` 是 Python 标准库中的一个模块，用于提供几个额外的数据结构，拓展了 Python 的内置数据类型（如 `dict`, `list`, `set`, 和 `tuple`）功能。`collections` 模块中的类和方法对于数据的高效存储和操作非常有用。

|**数据结构**|**说明**|
|---|---|
|`namedtuple`|用于创建具有命名字段的元组。|
|`deque`|双端队列（高效的双向插入和删除）。|
|`Counter`|可用于统计元素频率的哈希表。|
|`OrderedDict` (Python 3.7+ `dict` 支持顺序)|按插入顺序维护元素的字典（从 3.7 后普通 `dict` 也默认支持）。|
|`defaultdict`|带有默认值的字典（访问不存在的键时会返回默认值）。|
|`ChainMap`|将多个字典组合成为单个视图，形成一个“链式映射”。|
|`UserDict`, `UserList`, `UserString`|提供可继承类，用于重写和扩展字典、列表和字符串的功能。|
`namedtuple` 是一个工厂函数，用于创建具备命名字段的轻量级不可变对象。它可以像访问属性一样访问字段。
**附加功能：**
- 使用 `_fields` 属性查看字段名。
- 使用 `_asdict()` 方法将对象转为字典。
- 使用 `_replace()` 方法创建新实例（基于现有实例替换部分字段）。
`deque`（双端队列）支持在两端高效的插入和删除操作（相比于 `list` 在两端的操作效率更高）。
- 设置最大长度队列：当 `deque` 设置 `maxlen` 后，添加新元素将会自动移除多余的旧元素。
`Counter` 是一个多功能的字典，用于统计元素的出现次数。
`OrderedDict` 是一个按添加顺序保存键值对的字典。
`defaultdict` 是一种字典，在访问不存在的键时，会返回默认值（通过 `default_factory` 提供）。
`ChainMap` 将多个字典合并为单个可搜索的字典对象，支持优先级（类似作用域嵌套）。
`UserDict`, `UserList`, `UserString`这些类提供自定义字典、列表、字符串的基础。常用场景是继承这些类并重写核心功能。
